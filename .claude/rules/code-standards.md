\# Code Standards \& Best Practices



When reviewing any code or content:



\## Good Things to Look For

\- Clear variable and function names

\- Comments that explain WHY (not WHAT)

\- Code that's easy to read

\- Each function does one thing well

\- Tests to verify it works



\## Problems to Flag

\- Code without any comments

\- Very long functions (hard to read)

\- Confusing or unclear names

\- Code that's hard to understand

\- No tests for important functionality



\## When Reviewing

Always provide:

1\. What's wrong (if anything)

2\. Why it's a problem

3\. How to fix it

4\. What's done well (encouragement!)
